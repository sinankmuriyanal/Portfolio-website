ETL Data Engineering with Uber Trip Data and GCP products

Project Overview

Designed and implemented a complete ETL (Extract, Transform, Load) pipeline to process NYC Uber trip data using Google Cloud Platform (GCP) and Mage.ai. The goal was to transform raw datasets into structured, query-optimized tables for analysis and visualization.

Data Modeling & Architecture

Extracted trip records from NYCâ€™s publicly available dataset. Designed a star schema with a fact table (trip data) and multiple dimension tables for optimized storage and querying. Used Lucidchart to create a structured data model, ensuring minimal redundancy and faster queries.

Building the ETL Pipeline

Uploaded the raw dataset to Google Cloud Storage (GCS) and generated a private access URL. Deployed a virtual machine on Google Compute Engine to run the ETL process. Implemented an automated ETL pipeline in Mage.ai, leveraging prebuilt extractor, transformer, and loader blocks. Integrated custom Python scripts (developed in Jupyter Notebook) for advanced data transformations.

Data Storage & Querying

Loaded transformed fact and dimension tables into Google BigQuery for efficient querying. Conducted exploratory data analysis (EDA) using SQL to validate transformations and ensure data integrity.

Text Analysis for Sentiment and Readability Leveraging NLP & Python

Project Overview

This project involved web scraping, sentiment analysis, and text readability evaluation using Python and NLP techniques. By extracting textual data from a set of URLs, I aimed to analyze sentiment polarity, subjectivity, and readability metrics, enabling deeper insights into the structure and tone of the content.

Objectives

Extract article text from multiple URLs while filtering out unnecessary elements like headers and footers. Perform sentiment analysis to determine positive/negative scores, polarity, and subjectivity of the text. Assess readability using key metrics like Fog Index, syllable count, word complexity, and pronoun frequency. Generate structured outputs in Excel format for easy navigation and analysis.

Tools & Techniques Used

Web Scraping: Utilized BeautifulSoup to extract and clean textual content from web pages. NLP Techniques: Applied NLTK for stopword removal, tokenization, and complex word identification. Custom Algorithms: Developed functions to compute key metrics like: Polarity & Subjectivity (TextBlob) Syllable Counts & Complex Words (Custom logic) Fog Index & Readability Scores Personal Pronoun Identification Data Output: Structured Excel reports with clickable source links for quick reference.

Key Results & Insights

Successfully extracted and processed textual data from multiple web sources. Conducted sentiment analysis to gauge positive/negative tone and subjectivity in writing.
